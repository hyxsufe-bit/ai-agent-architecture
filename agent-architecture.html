03 检索先搜再答

RAG 的双管道 — 离线建索引，在线做检索。

大多数实用 Agent 背后不动声色的秘密：先搜，再答。 两条流水线，一件正事。

04 工具三者比较

Tool · MCP · Skill — 三种赋予 Agent 双手的方式。

函数调用近而便宜，MCP 远而通用，Skill 懒而节俭。 知其所需，方能择其所用。

维度 Function Call MCP Skill
调用方式 本地函数调用 远程标准协议（Anthropic） 本地渐进式加载
核心价值 扩展 LLM 执行能力 即插即用 · 跨应用复用 避免过多工具带来的 token 浪费
加载时机 全量注入 Prompt 全量注入 Prompt 先加载 metadata，用到才展开
适用场景 简单业务函数 跨厂商生态 复杂多步骤的技能包
代表实现 OpenAI function calling Claude Desktop · Cursor Claude Agent Skills
经验之规 基础六件套 bash / read / write / edit / grep / find 覆盖了 Agent 日常所需的八成以上。Vercel 的经验广为流传 —— 他们砍掉了八成工具，结果 text-to-SQL 的准确率从 80% 升到了 100%。
05 记忆短与长

短与长的结构 — 留下什么，压缩什么，遗忘什么。

短期记忆承载一段对话，长期记忆承载一个人。 在两者之间，住着一位压缩器 —— 决定谁是谁。

06 分工隔离之美

多智能体社会 — 一个主帅，几位斥候。

07 六则临别赠言

收官之语 — 六句话，带它回家。

纵使忘掉本册的其他一切，这六条顽固的心法， 依然足以护佑你走得更远。

  • 其 · 壹 Agent 之魂 Chatbot + Tools + Loop。除此之外，并无玄学。
  • 其 · 贰 检索为本 先搜再答。Embedding、向量库与重排器，才是真正在干活的三位。
  • 其 · 叁 少即是多 六件精挑细选的工具，胜过八十件花哨堆砌。克制本身就是一种功能。
  • 其 · 肆 记忆复利 要紧处逐字留存，其余压为恒久事实。唯此，记忆方能生息。
  • 其 · 伍 慎用分身 一个单 Agent 打磨到位，胜过三个仓促并行。多智能体不是万灵药。
  • 其 · 陆 最后一公里 护栏与可观测性并非可选项。生产级系统，从它们开始。
