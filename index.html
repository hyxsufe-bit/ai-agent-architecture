03 RAG先搜再答

RAG：先搜一把再回答 — 离线建好索引，在线只管查。

大多数好用的 Agent，本质上都在做同一件事：先搜索、再回答。 做法分两步 —— 平时建好索引，问的时候直接查。

04 工具三种形态横向对比

Tool、MCP、Skill 到底啥区别 — 给 Agent "装工具"的三种方式。

Function Call 是本地函数，简单直接；MCP 是远程协议，跨厂商通用； Skill 是本地包，按需加载省 token。知道各自适合什么，选起来就不纠结。

维度 Function Call MCP Skill
怎么调 直接调用本地函数 走远程协议（Anthropic 定的标准） 本地加载，按需展开
解决什么 给 LLM 加执行能力 让工具跨应用、跨厂商复用 工具太多时，省掉 token 浪费
加载方式 全量塞进 Prompt 全量塞进 Prompt 先只加载目录，用到才展开
适合场景 简单的业务函数 跨生态协作 步骤多、逻辑重的技能包
代表 OpenAI function calling Claude Desktop、Cursor Claude Agent Skills
一条经验 其实 bash / read / write / edit / grep / find 这六件基础工具， 已经能覆盖 Agent 八成以上的日常。Vercel 干过一件事 —— 砍掉 80% 的工具后， text-to-SQL 的准确率反而从 80% 提到了 100%。
05 记忆短期 + 长期

记忆是怎么存下来的 — 什么要留、什么要压缩、什么可以忘。

短期记忆记的是"这次聊了什么"，长期记忆记的是"这个人是谁"。 两者中间还夹着一个压缩器，决定哪些话要整段留着、哪些只保留要点。

06 多 Agent为什么要分身

多 Agent 协作是怎么回事 — 主 Agent 管大局，子 Agent 干脏活。

大部分场景一个 Agent 就够了。只有当它越干越乱、上下文被工具日志塞爆的时候， 才需要拆出几个"子分身"去隔离处理 —— 这不是为了多，是为了干净。

07 收尾带走这六句就够

六句话带走 — 其他全忘了也没关系，记住这六条。

如果整本手册只让你带走一页，那就是这一页。六句话，每句对应一个章节的结论。

  • 01 Agent 的本质 就是 Chatbot + 工具 + 循环，没别的玄学。
  • 02 RAG 的本质 先搜再答。干活的是 Embedding、向量库、重排器三件套。
  • 03 工具贵精不贵多 6 个基础工具比 80 个花哨工具好用得多，砍工具往往能提准确率。
  • 04 记忆要分层 重要的原样留着，其余压缩成要点。短期 + 长期，缺一不可。
  • 05 别急着上多 Agent 单 Agent 打磨到位，往往胜过三个 Agent 凑合着跑。
  • 06 上线前看护栏 Guardrails 和可观测性不是加分项，是及格线。
